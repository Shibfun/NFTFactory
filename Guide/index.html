Woofswap NFT Factory - User Guide

Introduction

Welcome to the Woofswap NFT Factory! This guide will help you create, manage, and mint NFTs on the Shibarium blockchain using our user-friendly interface. Follow the steps below to get started.

Prerequisites

  • MetaMask wallet installed in your browser (download here).
  • Shibarium network configured in MetaMask (chain ID: 109, RPC: https://rpc.shibariumchain.com).
  • Sufficient BONE (Shibarium native token) or ERC20 tokens for transaction fees and minting.
  • Access to the Woofswap NFT Factory interface.

Using the NFT Factory

1. Connect Your Wallet

  1. Open the Woofswap NFT Factory interface in your browser.
  2. Click the Connect Wallet button.
  3. MetaMask will prompt you to connect. Select your account and approve.
  4. Ensure you're on the Shibarium network. The interface will automatically switch if needed.
  5. Once connected, the button will show your wallet address (e.g., "Connected: 0x123...4567").

2. Create an NFT Contract

  1. In the Create New NFT Contract section, enter the following details:
    • NFT Name: Full name of your NFT collection (e.g., "My Awesome NFT").
    • NFT Symbol: Short symbol for your NFTs (e.g., "MANFT").
    • Max Supply: Total number of NFTs allowed (e.g., 10000).
    • Payment Type: Choose "BONE (Native)" or "ERC20 Token".
    • Payment Token Address: For BONE, itâ€™s set to "0x000...000". For ERC20, enter the token contract address.
    • Mint Price: Price per NFT in BONE or ERC20 tokens (e.g., 100).
  2. Click Create NFT Contract.
  3. MetaMask will prompt you to confirm the transaction. Approve it.
  4. Wait for the transaction to confirm. The new contract address will appear in the NFT Contract Address field.

3. View Your Created Contracts

  1. In the Your Created NFT Contracts (Latest 10) section, see the latest 10 NFT contracts you created.
  2. Each entry shows the NFT name, symbol, and contract address.
  3. Click the Copy button to copy a contract address for use elsewhere.

4. Set the Base URI for Your NFT Contract

  1. In the Manage NFT Contract section, enter or paste the NFT contract address.
  2. Enter the Base URI (e.g., "https://ipfs.io/ipfs/Qm.../") in the provided field.
  3. Click Set Base URI. Only the contract owner can perform this action.
  4. MetaMask will prompt you to confirm. Approve the transaction.
  5. Wait for confirmation. The Base URI will be set for all NFTs in the contract.

5. Mint an NFT

  1. In the Manage NFT Contract section, ensure the correct contract address is entered.
  2. Click Mint NFT.
  3. MetaMask will prompt you to confirm, showing the mint price (in BONE or ERC20 tokens).
  4. For ERC20 tokens, you may need to approve the contract to spend your tokens first.
  5. Wait for the transaction to confirm. The new NFT will appear in the Owned NFTs list.

6. View Your Owned NFTs

  1. In the Owned NFTs section, see the NFTs you own for the selected contract.
  2. Each NFT shows its ID (e.g., "NFT ID: 1").
  3. Click the View button to preview the NFT image (if the Base URI is set correctly).

Setting Up JSON Metadata (Base URI)

The Base URI is a URL pointing to a JSON file that defines metadata for all NFTs in your contract. This metadata includes the NFT's name, description, and image. Here's how to set it up:

1. Create the JSON Metadata File

Create a JSON file (e.g., `metadata.json`) with the following structure:

{
    "name": "Your NFT Name",
    "description": "Description of your NFT collection",
    "image": "https://ipfs.io/ipfs/Qm.../image.png"
}
  • name: Name of the NFT collection.
  • description: Brief description of the collection.
  • image: URL to the NFT image (e.g., hosted on IPFS).

Note: The Base URI is used for all NFTs in the contract, so this JSON applies to every NFT. If you need unique metadata per NFT, you must set up a server or use a dynamic IPFS folder structure (see advanced setups).

2. Upload the JSON and Image Files

You can host your JSON and image files on IPFS or other platforms. Here are two recommended options:

Option 1: IPFS (Using Pinata)

  1. Sign up for a free account at Pinata.
  2. Upload your image file (e.g., `image.png`) to Pinata. Copy the IPFS URL (e.g., "https://ipfs.io/ipfs/Qm.../image.png").
  3. Update your `metadata.json` with the image URL.
  4. Upload `metadata.json` to Pinata. Copy the IPFS URL (e.g., "https://ipfs.io/ipfs/Qm.../metadata.json").
  5. Use this URL as the Base URI in the NFT contract.

Option 2: NFT.Storage

  1. Sign up at NFT.Storage.
  2. Upload your image and JSON files via the web interface or API.
  3. Retrieve the IPFS URLs for your files.
  4. Use the JSON fileâ€™s URL as the Base URI.

3. Set the Base URI in the NFT Contract

Follow the steps in "Set the Base URI for Your NFT Contract" above to apply the Base URI to your NFT contract. Ensure the URL is publicly accessible.

Hosting the NFT Factory Interface

You can host the Woofswap NFT Factory interface (HTML file) on various platforms. Below are instructions for two popular options: GitHub Pages and Vercel.

Option 1: GitHub Pages

  1. Create a new repository on GitHub.
  2. Upload the `index.html` file (the NFT Factory interface) to the repository.
  3. Go to the repositoryâ€™s Settings tab.
  4. Scroll to the Pages section.
  5. Under Branch, select the main branch and set the folder to / (root).
  6. Click Save. GitHub will provide a URL (e.g., "https://yourusername.github.io/your-repo").
  7. Access the interface at this URL.

Option 2: Vercel

  1. Sign up for a free account at Vercel.
  2. Create a new project and connect it to your GitHub repository containing `index.html`.
  3. Vercel will automatically detect and deploy the HTML file.
  4. Once deployed, Vercel will provide a URL (e.g., "https://your-project.vercel.app").
  5. Access the interface at this URL.

Other Hosting Options

  • Netlify: Drag and drop `index.html` to Netlify Drop for instant hosting.
  • Firebase Hosting: Use Googleâ€™s Firebase Hosting for scalable deployment.

Troubleshooting

  • Wallet not connecting: Ensure MetaMask is installed and unlocked. Try refreshing the page.
  • Network error: Verify that MetaMask is on the Shibarium network (chain ID: 109).
  • Insufficient funds: Ensure you have enough BONE for gas fees and minting (or ERC20 tokens for minting).
  • Invalid Base URI: Check that the Base URI points to a valid JSON file and is publicly accessible.
  • NFT image not loading: Verify the image URL in your JSON metadata is correct and accessible.
